Guia dels **passos 1–3** del [pipeline a la homepage](/ca#process).

<h2 id="oauth">Login OAuth</h2>

Decap necessita un token de GitHub. Aquest stack usa un **gateway Express autoallotjat** (`gateway/`) al VPS.

Flux: `/admin` → gateway → GitHub OAuth → callback → token a Decap.

Variables: `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`, `AUTH_GATEWAY_URL`.

<h2 id="decap">Editar a Decap CMS</h2>

Col·leccions a `src/content/`. `config.yml` amb `base_url` substituït en el deploy.

<h2 id="git-commit">Commit a GitHub</h2>

Desa a `develop` (test) o `main` (producció). Contingut en git amb PRs normals.
